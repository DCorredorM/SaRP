Package Design

The package design is as follows. We have defined the following classes: 

Action
State
Actions 
States
Policy 
Policies
DPSEnumeration 
MDP
DPProblem

The last one is an abstract class, that contains the following abstract methods (functions): 

double immediateReward(State p, Action q, int stage);
Actions getActions(State p, int stage)
States getStates(int stage)
State T(State p, Action q, stage t)

Optionally, the user can override the following functions:

States getStates(int stage): The set of possible states at this stage. By default returns a common set of states.

double operation(double present, double future) : the operator between present and future rewards. By default it's sum.

double finalReward(State p). By default it is the optimal of immediate reward over all possible states.

If a user wants to solve a DP problem al she/he will have to do is to extend the class DPProblem, implementing the abstract funcions. It can be seen that there is a one to one correspondance between the theoretical and the computational framework, since all we want the user is to especify the forementioned elements of the DP Problem. This focus the user on the modelling, and leaves the computation to the internally defined functions.

Frequently the user might want to extend the classes Action and State, to define what the actions and states are. And also the class States and Actions, in order to define eficient numeration schemes.

Once the model is well defined, all the user will have to do is to call the method "solve()". After the problem has been solved, the user can read the solution printing "bestPolicy()", or with the variable "policies" obtained calling getPolicies().

In the stochastic case there are some slight modifications:

  • The user should extend the class MDP instead of DPProblem.
  • Instead of a transition function, the user must especify a transition probability function prob(p2,p1,q).
  • The user can override the method reachable(State p) that gives the set of reachable states from a given state. This is particularlly useful when the probabilities matrx is a disperse matrix, because this will guarantee that the algorithm only checks some few states at every step, ignoring those for which the probability is 0.

To fully understand each of this classes you should check the API (Aplication user Interface) at: http://www.isye.gatech.edu/ griano/DPS/doc/Package-Dp.html.
