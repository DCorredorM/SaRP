Problem Statement

As a simple example on how the code can be used, we solve a cow herd problem stated as follows: A farmer has 50 cows at t=0, and must decide at the end of each year whether to keep them or to sell them. The forecasted selling prices for the nest five years are: 10,15,25,7,10.

To solve this problem we proceed as follows. For each of this items the user can see the equivalence in the code attached at the end of this section.

  1. Define States: We will define the state as the number of cows that the farmer currently has. If we look closely to the formulation we will notice that if selling one cows is advisable, then selling all of them is advisable as well. However for illustration purposes we will ignore that fact. We define the class MyState as an extension of the class State, whith one field called caws. Notice that the number of possible states goes from 50 up to 378 at the last stage (year 5). This fact is reflected in our code, through an implementation of getStates() that return only the possible states for the given stage.
  2. Actions. At any state two possible actions can be take: keep or sell. Actually we could have also differentiated different actions for the number of caws sold.
  3. The reward obtained is nothing if we don't sell and price*caws if we sell them.
  4. The transformation will be T(p,q,k)= 1.5p if we don't sell and T(p,q,k)=0 if we sell.
